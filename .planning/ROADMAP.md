# Roadmap: Claude Agent Manager (CAM)

## Overview

CAM goes from blank Tauri 2 project to a shippable Windows desktop app in four phases. Phase 1 builds the Rust core — scanner, parser, toggler, groups, and the IPC command layer — before any UI exists. Phase 2 builds the complete React UI on top of that stable API surface: cards, sidebar, search, filters, status bar, and all toggle interactions. Phase 3 adds the file watcher for live external updates. Phase 4 produces the Windows installer. Every phase delivers a verifiable capability that is independently testable.

## Phase Details

### Phase 1: Foundation
**Goal**: The Rust backend is fully functional — it can scan directories, parse frontmatter, detect enabled/disabled state, group by prefix, and expose all operations through a typed Tauri IPC layer with correct permissions configured
**Depends on**: Nothing (first phase)
**Requirements**: SCAN-01, SCAN-02, SCAN-03, SCAN-04
**Success Criteria** (what must be TRUE):
  1. Running `cargo test` passes all unit tests for parser, scanner, toggler, and groups modules
  2. A test harness or Tauri devtools call to `get_agents` returns a typed `Vec<Agent>` with correct enabled/disabled state derived from `.disabled/` presence
  3. Calling the toggle command moves the target `.md` file into/out of `.disabled/` on disk and the filesystem state matches the command result
  4. A file with malformed or missing YAML frontmatter is returned with graceful fallback (filename as display name, `invalid_config: true`) rather than dropped or panicking
  5. TypeScript bindings generated by `tauri-specta` compile without errors and match Rust command signatures

### Phase 2: React UI Core
**Goal**: A user can open the app, see all their agents/skills/commands as cards with correct visual state, navigate between sections, search and filter, toggle any item or group on/off with immediate feedback, and see error states for bad configs and missing directories
**Depends on**: Phase 1
**Requirements**: DISP-01, DISP-02, DISP-03, DISP-04, DISP-05, DISP-06, DISP-07, DISP-08, TOGL-01, TOGL-02, TOGL-03, NAV-01, NAV-02, NAV-03, NAV-04
**Success Criteria** (what must be TRUE):
  1. User sees agent cards with name, description, color dot, and visual disabled state (opacity 0.5 + grey toggle) that accurately reflects the current `.disabled/` filesystem state on launch
  2. Clicking a toggle flips the UI state within 200ms, the underlying file moves on disk, and on error the UI reverts to the previous state with a toast notification
  3. User can switch between Agents, Skills, and Commands sections via sidebar and each section loads its correct content independently
  4. User can type in the search box or press Ctrl+F to focus it and the card list filters in real-time by name and description; filter pills (All / Enabled / Disabled) update their counts dynamically
  5. User sees the status bar showing the scan path, count of enabled agents (green dot) and disabled agents (grey dot), and app version

### Phase 3: Live Updates
**Goal**: External filesystem changes (an agent file moved, renamed, or added outside the app) are automatically reflected in the UI within 500ms without manual refresh, and the app's own toggle operations do not trigger spurious re-renders
**Depends on**: Phase 2
**Requirements**: LIVE-01, LIVE-02, LIVE-03
**Success Criteria** (what must be TRUE):
  1. Moving a `.md` file into or out of `.disabled/` using the OS file explorer causes the corresponding card to update its toggle state in the UI within 500ms with no user action
  2. Performing a toggle inside the app does not cause the UI to flicker, double-update, or lose the optimistic state due to a watcher event from the app's own operation
  3. Opening the app and waiting for React to fully mount before watcher events are processed — no missed or duplicated events during the startup window

### Phase 4: Distribution
**Goal**: The app ships as a Windows installer that a developer can download, run, and have a working CAM installation without manual steps
**Depends on**: Phase 3
**Requirements**: DIST-01, DIST-02
**Success Criteria** (what must be TRUE):
  1. Running `cargo tauri build` produces a valid `.msi` or NSIS `.exe` installer file for Windows
  2. The installer includes a custom app icon (not the default Tauri icon) and correct version metadata (product name, version, publisher) visible in Windows Add/Remove Programs
